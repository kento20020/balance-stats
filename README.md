# balance-stats

ゲームバランス検証のための統計ライブラリと、
LLMレビュー比較（Claude Fable 5 vs Opus 4.8）の一次資料。

## このリポジトリについて

Zenn記事の補足資料です。
https://zenn.dev/beckento/articles/55fb157f9c242c

ゲーム設計書のバランス検証手法に潜んでいた統計的誤謬
（「有意差なし＝差なし」と判定してしまう構造）を、
2つのモデルがどうレビューしたかを比較しました。

## ファイル構成

| パス | 内容 |
|---|---|
| `docs/game_design_document_v0_5.md` | レビュー対象の設計書（v0.5、開発途中） |
| `logs/fable5.md` | Claude Fable 5 のレビュー応答全文 |
| `logs/opus4_8.md` | Claude Opus 4.8 のレビュー応答全文 |
| `INTEGRATION.md` | シミュレーションハーネスへの組み込みガイド |
| `balance_stats.py` | Fable 5 が実装した統計ライブラリ（numpy/scipyのみ依存） |

## ログについて

- 両モデルに同一の設計書・同一のプロンプトを入力
- Q1: 「内容を添削して修正案を教えて。」
- Q2: 「統計的な確認方法は問題ないか確認して。」（同一チャット内）
- 各モデル1会話ずつ。応答は無編集（整形のみ）

## ライセンス / 注意

- 設計書は開発途中のドラフトであり、記事で扱った箇所以外にも未修正の問題を含みます
- ログは一次資料としてそのまま公開しています（n=1の比較であり性能ベンチマークではありません）
